# AI Star Studio — Claude Design 设计 brief（前端落地参照）

> 来源：用户在 claude.ai/design 用「Design Components」建的 `AI Star Studio.dc.html`（深色制片棚主题，紫+青双主色）。本文件是其 §1–§4 规格的留档，作为把 `prototype/` 前端对齐到该设计的依据。后端契约（M1/S1/S2 API）不变。

## §0 产品定位（不变量）
以「艺人」为中枢的专业级 AI 创作工具，**不是经营游戏**。先对话访谈捏出虚拟艺人，再用 5 条专业制片级生产线产出：对话陪伴、写真/视频、音乐、访谈成片、短剧。
- 艺人为中枢：顶部艺人选择器切换后，所有生产线锁定同一张脸/声线（一致性参考包）。
- 专业感非玩具感：分阶段可控工作流（stepper）、单点重抽、资产管理、质量档位、生成前成本闸门。
- 视觉基调：深色制片棚，紫 #7c5cff + 青 #26d6c4 双主色，克制、专业、有质感。

### 排版
- 字体栈：`"PingFang SC","Microsoft YaHei",-apple-system,"Segoe UI",Roboto,sans-serif`
- 字号阶：H1 21 / H2 15 / 卡片标题 14.5 / 正文 14 / 次要 13 / 标签 11–12 / 微标签 10.5
- 字重：正文 400，强调/标签 600–700，标题/价格 800
- label 态：11.5px、letter-spacing .3px、uppercase、--ink-3

### 间距 / 圆角 / 阴影 / 布局
- 圆角：卡片 14px(--r)、控件 9px(--r-sm)、chip/pill 30px（全圆）
- 卡片内边距 18px；视图内边距 24–26px；栅格间距 16px
- 阴影（浮层/模态）：`0 14px 40px rgba(0,0,0,.45)`；主按钮辉光 `0 8px 20px #7c5cff22`
- 三栏外壳：左导航 **236px 固定** + 主区自适应；主区内顶栏 + 滚动视图
- 栅格：g2/g3/g4 等分；split = 1.15fr / 0.85fr（主操作+侧信息）；≤1100px 时 g3/g4 降 2 列、split 降单列

## §2 组件清单
- **Sidebar**：Logo 块（conic 渐变 logo + 标题/副标题）+ 分组（工作台/生产线/系统）+ 导航项（图标+标签+里程碑 tag）+ 底部 Provider 状态条。激活态：左渐变高亮 + 3px 紫色指示条。
- **Topbar**：面包屑 + 艺人选择器（头像+艺名+人设，下拉切换）+ 新建按钮。
- **ArtistPicker** 下拉：艺人项（头像/艺名/人设）+「创设新艺人」入口。
- **Button**：主（渐变紫+辉光）/次（描边）/幽灵/小/图标；hover 上浮 1px。
- **Chip**（可选标签，默认/激活紫底）；**TierToggle**（双档分段：草稿↔发行 / 开↔关）。
- **Input/Textarea/Select**：深底，聚焦紫描边 + 3px 辉光；Field 含大写 label。
- **Pill**（状态徽标：里程碑/待确认/AI 已补全）；**Tag**（只读小标签）。
- **Stepper**：done(绿勾)/cur(紫高亮+辉光)/pending(灰)；音乐 4 段、访谈 5 段、短剧 7 段。
- **JobCard**：spinner + 阶段进度条 + jobId + 阶段说明 + 暂停/重试。
- **CostGateModal**：任务/档位/艺人/预估成本（大号金色）+ 确认才入队。
- **MediaTile**：写真 3:4 / 视频 9:16；**锁脸徽标（青色）**、收藏/重抽悬浮操作、底部元数据条。
- **ProfileCard**：身份 / 真实感内核 / 一致性参考包 三段，字段可编辑。
- **HealthStat** 三态行；**KV** 键值表；**Toast** 轻提示。

## §3 逐页规格（页面 · 里程碑 · 核心交互 · 关键状态）
- **艺人创设（首屏）S1**：左访谈对话（一次一问）→「生成档案」→ 右可编辑档案卡 →「确认创建」/「出定妆照」；下方艺人卡片网格。状态：访谈中、档案草稿(AI 已补全)、定妆照生成中、空态、字段校验错误。
- **对话陪伴室 S2**：in-character 流式打字回复；侧栏亲密度/情绪进度、长期记忆、主动消息开关。状态：流式中、对方 typing、记忆为空。
- **写真/视频 S3**：锁脸提示 + 可控参数（景别/比例/变体/档位）+ 生成；画廊筛选（全部/写真/视频/收藏）。状态：生成中(JobCard)、单图重抽、收藏态、画廊空态。
- **音乐工坊 S4**：两段式——可编辑作曲蓝图（确认前）→ 渲染；作品库可播放/带入短剧。状态：蓝图待确认、重抽蓝图、渲染中、空态。
- **访谈成片 S5**：5 阶段管线 + 双人画面合成；阶段逐步更新、可暂停/重试。状态：script/audio/visual/subtitle/final 五态、阶段失败带原因。
- **短片/短剧 S6**：7 阶段管线；分镜板单格重抽；低成本↔高质量档；逐集/整剧成本闸门。状态：分镜生成中、单格重抽、整剧批量确认、选角定妆。
- **能力健康 M1**：6 Provider 三态、降级链说明。**成本账本 M1**：月度统计卡 + 用量明细表。**设置 M1**：6 能力 key 录入（仅服务端、no-store）。**设计系统**：tokens + 组件画廊。

## §4 落地约束
- 不引入经营游戏元素（资金/行动点/热搜/危机/日结）。
- 一致性优先：任何图像/视频/配音 UI 都要可见「锁脸锁声」标识（青色徽标 + 艺人名），体现 PS-1。
- 专业 = 可控：每条重媒体线呈现分阶段、可单点重抽、有质量档位、生成前确认成本——不做「一键黑盒」。
- 诚实边界：keyless 现状用 unconfigured（warn 黄）而非「报错红」；真实出片需配 key。
- 可落地：实现到 `prototype/` 前端（**原生 http、零 npm 依赖、单文件风格、不依赖重型前端框架**）。设计的 React 原型仅作视觉参照，落地用原生 JS。

## 落地策略（分阶段，对齐 §0「不要一次铺全部页面」）
1. **P1 设计系统 + 外壳 + 已就绪页**：新 styles.css（§1 tokens + §2 组件）+ 三栏外壳（sidebar/topbar/artist-picker/视图路由）+ 把现有可用页对齐设计：艺人创设(S1)、对话陪伴(S2)、能力健康、成本账本、设置、仪表盘。后端已就绪，纯前端重做。
2. **P2 写真/视频(S3)**：通义万相(DashScope)图像/视频适配器（区域可用）+ 写真控制台 + 画廊 + 视频 job。
3. **P3 音乐(S4) / 访谈(S5) / 短剧(S6)**：各自后端 + 页面，按里程碑推进。

每阶段沿用 TDD + 子代理双评审 + 合并的纪律；后端契约不破坏（M1/S1/S2 API 已固化）。
