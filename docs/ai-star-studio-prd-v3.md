## v3 相对 v2 的三个根本变化

1. **模型与平台全面开放**：不绑定 OpenRouter。引入「能力网关」抽象层，每个 AI 能力可独立接入任意平台（Anthropic/OpenAI/Google 直连、阿里云、火山引擎、Kling、Suno、ElevenLabs、Fal.ai 等聚合站），OpenRouter 降级为众多 Provider 之一。
2. **彻底去本地化**：移除对本机显卡与本地模型服务（ACE-Step、ComfyUI WAN）的依赖，音乐/视频全部改接专业云端 API。游戏在任何一台普通电脑上双击即玩。
3. **新增 AI 短剧**：完整的竖屏短剧生产管线（剧本 → 选角 → 分镜 → 视频 → 配音字幕 → 成片 → 上线经营反馈），成为与音乐、写真、访谈并列的第四条内容生产线，也是商业想象力最大的一条。

## 1. 产品定位

**一句话**：玩家经营一家 AI 娱乐公司，旗下虚拟艺人的歌曲、写真、视频、访谈节目、短剧全部由专业 AI 真实生产——这是一家「真的能出片」的虚拟经纪公司。

**核心体验承诺**（继承 v2，新增第 4 条）：

1. 艺人是活的：实时生活线、可对话、路线分化。
2. 内容是真的：每种内容都是可消费的真实媒体资产。
3. 娱乐圈是转的：热搜、粉丝、收入随世界时钟与玩家行动演化。
4. **作品是成体系的**：艺人可以靠一部短剧爆红，主题曲来自音乐工坊，宣传照来自写真中心，宣发访谈来自访谈节目——四条生产线互相喂料，构成完整的造星工业。

## 2. 架构需求：能力网关（最重要的架构变更）

### 2.1 设计

```
浏览器（prototype/ 静态前端，零构建）
   │
Node 服务器 server.js（端口 3100）
   │
能力网关 AI Gateway（capability → provider 路由）
   ├── 文本类 Provider 适配器：Anthropic 直连 / OpenAI 直连 / Google Gemini 直连 /
   │                         阿里云 DashScope / 火山方舟 / OpenRouter（聚合兜底）
   ├── 图像 Provider：Gemini Image 直连 / 即梦(火山) / Fal.ai / OpenRouter
   ├── 视频 Provider：Kling 官方 / Veo(Gemini API) / 即梦 Seedance(火山) /
   │                 MiniMax 海螺 / Fal.ai / Replicate
   ├── 音乐 Provider：Suno 官方 / Mureka / MiniMax Music / Lyria(Gemini)
   ├── 语音 Provider：CosyVoice(DashScope) / ElevenLabs / MiniMax TTS
   └── 转写 Provider：DashScope ASR / OpenAI Whisper
```

### 2.2 需求条款

- **GW-1 能力为单位路由**：系统定义标准能力清单（chat / content / world / plan / image / video / music / tts / asr / drama-script / storyboard）。每个能力独立配置 `provider + model + 参数`，互不影响。
- **GW-2 Provider 适配器协议**：每个平台实现统一适配器接口（认证、请求构造、响应归一化、错误归一化、任务轮询）。新增一个平台 = 新增一个适配器文件，不改业务代码。
- **GW-3 配置驱动**：路由配置集中在 `config/ai-providers.json`（或等价 env 组），格式示例：
  ```json
  {
    "chat":   { "provider": "anthropic", "model": "claude-sonnet-4-6" },
    "music":  { "provider": "suno",      "model": "v5" },
    "video":  { "provider": "kling",     "model": "kling-v3-std",
                "fallback": [{ "provider": "volcano", "model": "seedance-2.0" }] }
  }
  ```
  换平台/换模型只改配置，零代码改动，重启生效（热加载为加分项）。
- **GW-4 跨平台 fallback**：失败重试不再局限于同平台换模型，支持跨平台降级链（如 Kling 官方失败 → 火山 Seedance → Fal.ai）。
- **GW-5 密钥管理**：每个 Provider 独立 key（`ANTHROPIC_API_KEY`、`SUNO_API_KEY`、`KLING_ACCESS_KEY` 等），全部仅存服务端 `.env`；未配 key 的 Provider 在健康面板显示「未接入」而非报错。
- **GW-6 统一异步任务模型**：所有重媒体生成（视频/音乐/短剧）走统一 job 队列（提交即返回 jobId、阶段进度、持久化、重启恢复），无论底层平台是同步还是轮询式。
- **GW-7 用量与成本账本**：每次调用记录 provider/model/耗时/用量/估算成本到 `logs/ai-usage.jsonl`，游戏内可查看本周期 AI 成本（同时作为经营游戏的「制作成本」数值来源——真实成本即游戏成本，天然平衡）。
- **GW-8 OpenRouter 定位**：作为文本与图像的聚合兜底渠道保留，不再是必经之路。

### 2.3 推荐默认路由（接入时以各平台现行 API 文档为准）

| 能力 | 首选 | 备选 | 说明 |
| --- | --- | --- | --- |
| 艺人对话/内容包/剧本 | Anthropic 直连 claude-sonnet-4-6 | OpenRouter 同款 | 人设一致性与中文创作；直连省聚合层加价与代理依赖 |
| 世界刷新/活动规划 | 阿里云 qwen3.7-plus（或 qwen-flash 档） | 火山豆包 | 高频低成本 |
| 写真/分镜图 | Google Gemini 3 Pro Image 直连 | 即梦 4.0（火山）/ Fal.ai | 人像一致性；分镜需角色参考图输入能力 |
| 日常/短剧/访谈视频 | Kling 官方 API（图生视频，竖屏） | Veo 3.1（Gemini API）/ Seedance（火山）/ MiniMax | 专业级出片，远超本地 5B 模型 |
| 歌曲 | **Suno 官方 API** | Mureka / MiniMax Music / Lyria | 带人声完整歌曲的行业标准，替代 ACE-Step |
| TTS | CosyVoice v3（DashScope） | ElevenLabs（多语种）/ MiniMax | 已验证；短剧多角色配音需音色库 |
| ASR | qwen3-asr-flash | Whisper | 已验证 |

## 3. 去本地化需求

- **CL-1** 系统不得依赖本机 GPU、不得要求本机部署任何模型服务。最低运行环境：能跑 Node ≥18 的普通电脑。
- **CL-2** ACE-Step（8001）与 ComfyUI（8188）从必需依赖中移除；相关代码降级为可选 Provider 适配器（`provider: "local-acestep"` / `"local-comfyui"`），默认关闭，供有 GPU 的玩家自选省钱。
- **CL-3** ffmpeg 保留为本机剪辑工具（纯 CPU、免费、无模型），负责长视频/短剧的字幕烧录与时间线拼接；提供云端剪辑（如 Shotstack 类服务）作为可选适配器，不作默认。
- **CL-4** `start_all.bat` 简化为：启动游戏服务器 + 打开浏览器。不再拉起 GPU 服务。
- **CL-5** 健康面板改为探测「已配置的云端 Provider」：key 是否存在、连通性、（可选）余额查询；并区分三态：在线 / 故障 / 未接入。
- **CL-6 成本透明**：每次重媒体生成前展示预估成本并需玩家确认（短剧/MV 等批量任务展示整批预估）。参考量级（接入时校准）：写真 $0.03–0.15/张、视频 $0.3–1.5/10 秒、整首歌 $0.2–0.8、短剧一集（1–2 分钟）低配 $1–3、高配 $5–15。

## 4. 功能需求

3.1–3.9 模块（公司/艺人/通告/写真/音乐/访谈/热搜/危机/健康监控）继承 v2 的 FR-101 至 FR-902，仅以下变更：

- **FR-402v3** 日常视频改云端视频 API（图生视频，艺人定妆照为参考帧）。
- **FR-502v3** 歌曲渲染改 Suno 类云端 API：输入蓝图（歌名/曲风/BPM/歌词/性别声线）→ 输出完整带人声歌曲（2 分钟+），不再区分"45 秒 clip/正式版"的本地渲染档位,改为「试听版（低成本快速）/ 发行版（高质量）」两档。
- **FR-604v3 / FR-605v3** 访谈视频镜头与长片画面改云端视频 API；TTS+字幕+拼接管线不变（ffmpeg）。

### 3.10 AI 短剧工坊（新模块，FR-1000 系列）

短剧是「竖屏、单集 1–3 分钟、强钩子、连载」的内容形态，也是艺人造星的核心载体。

**A. 立项与剧本**

- **FR-1001 立项**：选题材（赘婿逆袭/甜宠/悬疑/穿越/职场等模板 + 自定义）、集数（MVP 支持 3–12 集）、单集时长档（60s/120s/180s）、主演（旗下艺人，注入人设与视觉档案）。
- **FR-1002 剧本生成**：AI 产出剧集企划：全剧大纲、分集剧本（场景/动作/对白/情绪标注）、每集结尾钩子、角色表（主演=艺人，配角由 AI 生成人设）。剧本为结构化 JSON，支持逐集重写与人工编辑。
- **FR-1003 选角与定妆**：为每个 AI 配角生成人设卡与定妆照；主演使用艺人视觉档案,全剧保持形象一致。

**B. 制作管线（每集一个异步 job，阶段可视）**

- **FR-1004 分镜**：每场景生成分镜图（角色参考图输入保证一致性），玩家可重抽单格分镜。
- **FR-1005 视频生成**：分镜图 → 图生视频片段（竖屏 9:16），按场景时长切分；提供两档：
  - 低成本模式：静帧 + 运镜（ken burns）+ 配音，视频 API 用量减 80%；
  - 高质量模式：全片段 AI 视频。
- **FR-1006 配音与字幕**：对白按角色分配 TTS 音色（主演用艺人声线档案），生成 SRT 并烧录。
- **FR-1007 成片合成**：片段 + 配音 + BGM（可选用音乐工坊产出作为主题曲）→ 单集 MP4；全剧合集页可连播。
- **FR-1008 成本闸门**：每集生成前展示预估成本，整剧批量生成需逐集或一次性确认（见 CL-6）。

**C. 上线与经营反馈**

- **FR-1009 上线发行**：短剧按集上线进入世界引擎：生成播放量曲线、热搜话题、粉丝评论、媒体评价；钩子质量与艺人热度影响数据。
- **FR-1010 收入与造星**：播放分账计入公司资金；主演热度/粉丝/商业价值随剧集表现变化；爆款触发「艺人爆红」事件链（商务邀约、黑粉、续集压力）。
- **FR-1011 续作决策**：周期结算时基于数据给出续季/换题材/换主演建议，构成长线经营循环。

## 5. AI 能力需求

- **AI-1v3** 能力路由集中于 `config/ai-providers.json`（取代 server.js 常量区散布），见 GW-3。
- **AI-2** 玩家可感知内容禁用不可控聚合路由（openrouter/auto 类）。
- **AI-3v3 成本红线**：文本类一周 < $2；媒体类按次确认（CL-6），游戏内成本账本可查（GW-7）。
- **AI-4** 单 Provider 失败必须走跨平台降级链（GW-4），全链失败才报错并附原因与修复提示。
- **AI-5** 人设档案（性别/声线/视觉/性格）必须注入所有生成（含短剧分镜、配音、剧本中的主演台词风格）。
- **AI-6** 内容安全：全部 SFW；不得生成真实公众人物（访谈嘉宾公开照参考除外）；短剧题材模板内置红线过滤。
- **AI-7 一致性资产**：每位艺人维护「一致性参考包」（定妆照集 + 声线样本 + 视觉描述），所有图像/视频/TTS 生成必须引用，这是跨四条内容线形象统一的技术基础。

## 6. 数据与持久化

继承 v2 DR-1～DR-4，新增：

- **DR-5** 短剧项目（剧本/分镜/片段/成片/经营数据）持久化为独立项目目录 `prototype/generated/drama-<id>/`，含 project.json 全量状态，支持断点续作。
- **DR-6** AI 用量账本 `logs/ai-usage.jsonl`（GW-7）。

## 7. 非功能需求

- **NFR-1v3 性能**：聊天 ≤ 8s；写真 ≤ 60s；视频单段 ≤ 10min（云端排队波动须在 UI 呈现进度）；短剧单集（低成本模式）≤ 15min。
- **NFR-2v3 可靠性**：任一 Provider 故障不影响其它能力；健康面板 10s 内反映状态。
- **NFR-3 可观测**：全部 AI 调用失败入 `logs/` 结构化日志（含 provider/model/requestId/耗时）。
- **NFR-4 安全**：key 仅服务端；`.env` 不入库；路径穿越防护；`no-store`。
- **NFR-5v3 环境**：任意可跑 Node ≥18 的 Windows/macOS 设备；无 GPU 要求；需可访问所配 Provider 的网络出口（代理配置per-provider 可选）。

## 8. 运行与部署

- **OR-1v3** `start_all.bat` = 游戏服务器 + 浏览器，3 秒内可玩。
- **OR-2v3** 端口仅 3100；7890 代理仅在某 Provider 配置了 proxy 时使用。
- **OR-3** `npm run smoke` 覆盖全部文本端点 + 各已配置 Provider 的连通性探测；`npm run check` 通过为发布门槛。

## 9. 迁移计划与路线图

### M1 —— 能力网关与去本地化（架构地基，先行）
1. 抽出 Provider 适配器层（GW-1～GW-5），现有 OpenRouter/DashScope 调用迁入适配器。
2. 接入直连 Provider：Anthropic、Google Gemini（文本+图像）。
3. 音乐切换 Suno 类 API（FR-502v3），视频切换 Kling/Veo/Seedance（FR-402v3、604/605v3）；ACE-Step/ComfyUI 降级为可选适配器（CL-2）。
4. 健康面板与 start_all.bat 改造（CL-4、CL-5）；用量账本（GW-7）。

### M2 —— AI 短剧 MVP
5. FR-1001～1008：3 集 × 60 秒、低成本模式优先、单部短剧全管线跑通。
6. FR-1009～1011:上线数据与造星反馈接入世界引擎。

### M3 —— 经营循环补完（继承 v2 P0）
7. 行动点硬约束与日结仪式;周结算 S–D 评级;危机倒计时。
8. 短剧/歌曲/写真的制作成本与收入用真实 AI 成本驱动（GW-7 数据回流）。

### M4 —— 体验与留存（继承 v2 P1/P2）
9. 聊天流式输出;全媒体统一"生成中"卡片;模型与耗时展示。
10. 路线分化数值生效;存档系统;周报分享卡;自定义艺人。

### 验收基线变更
v2 第 9 节基线中与本地 GPU 绑定的三项（ACE-Step 43s、ComfyUI 2.5min、start_all 拉起 GPU 服务）随 M1 完成后由对应云端指标取代；其余基线（13 端点、零控制台报错、健康面板、写真）继续有效，不得劣化。
